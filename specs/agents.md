# AGENTS.md Spec

Spec version: 1.0
Source: https://agents.md/

## Overview

`AGENTS.md` is an open-standard Markdown file that provides AI coding agents
with project-specific context, operational guidance, and constraints. It is
stewarded by the Agentic AI Foundation (AAIF) under the Linux Foundation.

Think of it as a "README for agents" — a predictable location for information
that is not easily inferable by an agent or that would clutter a
human-focused `README.md`.

## Three Distinct Files in Nydus

| File | Location | Audience | Maintained By |
|------|----------|----------|---------------|
| Repo-level | `/AGENTS.md` | Coding agents working on Nydus | Manual |
| Per-egg | `AGENTS.md` inside `.egg` archive | Agent/developer hatching this egg | Generated at spawn |
| Hatched agent's | Target workspace `AGENTS.md` | The agent itself at runtime | Generated by hatcher |

The hatched agent's `AGENTS.md` already works today — it is not part of
this standards integration work.

## Recommended Content

Effective `AGENTS.md` files focus on non-inferable details:

- **Commands**: Full executable commands (build, test, lint) with flags
- **Conventions**: Coding style, naming patterns, architectural constraints
- **Boundaries**: "Always do", "Ask first", "Never do" rules
- **Project Context**: One-sentence project description, package manager

## Best Practices

- **Progressive Disclosure**: Keep the root file minimal (150–200 lines).
  Link to domain-specific files (e.g. `docs/TESTING.md`) to avoid token bloat.
- **Hierarchy**: Agents read the `AGENTS.md` closest to the file being
  edited. Use nested `AGENTS.md` in subdirectories for package-specific
  overrides.
- **Avoid Redundancy**: Don't duplicate information agents can discover on
  their own (codebase architecture, dependency lists).

## Per-Egg AGENTS.md Template

The per-egg `AGENTS.md` is a deployment runbook generated from egg data:

| Section | Source |
|---------|--------|
| Prerequisites | `manifest.min_nydus_version`, `manifest.agent_type` |
| Required Secrets | `SecretsModule` — name, kind, `required_at_hatch` |
| Hatch Command | Templated from `agent_type` |
| MCP Servers | `McpModule.configs` — server names, command, env |
| Verification Steps | Per `agent_type` conventions from `AGENT_SPEC.md` |
| Conventions | Per `agent_type` from `AGENT_SPEC.md` |

Template-based fallback produces a structured, usable document. LLM-enhanced
version takes the same inputs and produces better prose (optional).
