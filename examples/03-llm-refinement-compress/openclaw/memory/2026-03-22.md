Helped debug a networking issue — pods in EKS couldn't reach an RDS
instance in a different VPC. Turned out the VPC peering was set up
but the route tables weren't updated. Classic mistake. Walked through
the fix and recommended they document their network topology.

Also had a conversation about serverless vs containers for a new
internal tool. The tool has low traffic (maybe 100 requests/day) but
needs to connect to a VPC-internal database. Recommended Cloud Run
with a VPC connector over Lambda because the startup latency matters
less than the simpler deployment model for their team size.
